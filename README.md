# 📊 Transaction Q&A Assistant

A Streamlit application that lets users query their bank transaction statements using natural language. Built with a LangChain ReAct agent backed by GPT-4o-mini, it reasons across multiple tool calls to answer complex financial questions — with per-user login, conversational memory, and persistent chat logs.

---

## Features

- **Secure login** — static credentials per user stored in `.env`; each user sees only their own statements
- **Agentic Q&A** — LangChain ReAct agent with 5 analysis tools; reasons across multiple steps to answer complex questions
- **Conversational memory** — retains last 3 exchanges so follow-up questions resolve naturally (e.g. *"What about that month?"*)
- **Multi-month statements** — combined Jan–Mar 2025 PDF per user (33 transactions), generated by `sample_statements_pdf_generater.py`
- **Chat session logging** — every message saved to `logs/{user}_chat_history.jsonl` in industry-standard JSONL format
- **No vector database** — pure pandas operations; no embeddings or retrieval pipeline needed

## Architecture

```
┌─────────────────────┐
│   Login Page        │  Validates USER{n}_ID / USER{n}_PASSWORD from .env
└────────┬────────────┘
         │ authenticated
         ▼
┌─────────────────────┐
│   PDF Loader        │  pdfplumber → pandas DataFrame (user-scoped)
└────────┬────────────┘
         │ DataFrame
         ▼
┌─────────────────────────────────────────┐
│           LangChain ReAct Agent         │
│           (GPT-4o-mini)                 │
│                                         │
│  Tools:                                 │
│  ├── get_all_transactions()             │
│  ├── filter_by_month(month)             │
│  ├── summarize_by_type(month?)          │
│  ├── top_expenses(n, month?)            │
│  └── compare_months(month1, month2)     │
│                                         │
│  Memory: last 3 exchanges passed as     │
│  message history on every invoke        │
└────────┬────────────────────────────────┘
         │ answer
         ▼
┌─────────────────────┐
│   Streamlit Chat UI │  Renders history + reasoning trace expander
└────────┬────────────┘
         │ log each message
         ▼
┌─────────────────────┐
│  JSONL Chat Logs    │  logs/{username}_chat_history.jsonl
└─────────────────────┘
```

## Setup

### 1. Clone and create environment

```bash
conda env create -f environment.yml
conda activate transaction-qa
```

Or with pip:

```bash
pip install -r requirements.txt
```

### 2. Configure `.env`

```env
OPENAI_API_KEY=sk-...

USER1_ID=john_doe
USER1_PASSWORD=John@123
USER1_NAME=John Doe

USER2_ID=jane_smith
USER2_PASSWORD=Jane@456
USER2_NAME=Jane Smith
```

Add more users by incrementing the number (`USER3_*`, etc.).

### 3. Generate sample PDF statements

```bash
python sample_statements_pdf_generater.py
```

This creates one combined 3-month PDF per user in `data/`:
- `data/john_doe_statement_2025.pdf`
- `data/jane_smith_statement_2025.pdf`

### 4. Run the app

```bash
streamlit run app.py
```

## Agent Tools

| Tool | Description |
|---|---|
| `get_all_transactions()` | Overview — row count, date range, sample rows |
| `filter_by_month(month)` | All transactions for a given month |
| `summarize_by_type(month?)` | Debit / credit totals and counts |
| `top_expenses(n, month?)` | Top N highest debit transactions |
| `compare_months(month1, month2)` | Side-by-side debit/credit breakdown |

The agent calls tools in sequence as needed. For example, *"Which month had the highest spending and what were the top 3 costs?"* triggers `summarize_by_type` for each month, identifies the answer, then calls `top_expenses` — all without user intervention.

## Chat Logging

Every message is appended to `logs/{username}_chat_history.jsonl` as the conversation happens. Each line is one event:

```jsonl
{"session_id": "20260422_204500", "user_id": "john_doe", "pdf": "john_doe_statement_2025.pdf", "timestamp": "2026-04-22T20:45:32", "role": "user", "content": "Which month had the highest spending?"}
{"session_id": "20260422_204500", "user_id": "john_doe", "pdf": "john_doe_statement_2025.pdf", "timestamp": "2026-04-22T20:45:35", "role": "assistant", "content": "March had the highest debit spending at $2,393.79."}
```

Sessions are identified by a `session_id` timestamp generated when a PDF is loaded. Logs are gitignored and never committed.

To load logs into pandas for analysis:

```python
import pandas as pd
df = pd.read_json("logs/john_doe_chat_history.jsonl", lines=True)
```
